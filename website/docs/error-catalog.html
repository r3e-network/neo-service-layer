Error Classification System

Error Code Format

All error codes follow the pattern: [SERVICE]_[NUMBER]

  • SERVICE: 3-8 character service identifier
  • NUMBER: 3-digit sequential number (001-999)
Examples
AUTH_001 - Authentication service error #1 STORAGE_045 - Storage service error #45 CHAIN_123 - Blockchain service error #123

Severity Levels

Critical Service down, data loss risk, security breach
High Major functionality affected, performance degraded
Medium Feature limitation, recoverable errors
Low Minor issues, warnings, edge cases

Response Codes

HTTP status codes used alongside error codes:

  • 400: Client errors (validation, parameters)
  • 401: Authentication required
  • 403: Permission denied
  • 404: Resource not found
  • 429: Rate limit exceeded
  • 500: Internal server errors
  • 503: Service unavailable

General Troubleshooting Guide

Step-by-Step

Error Resolution Flowchart

Follow this systematic approach to diagnose and resolve errors:

1
Identify the Error
Collect the full error message, error code, timestamp, and request ID. Check logs for additional context.
2
Check Service Status
Visit the status page to verify if there are ongoing incidents or maintenance affecting the service.
3
Verify Authentication
Ensure API keys are valid, not expired, and have proper permissions for the requested operation.
4
Validate Request Parameters
Check request format, required fields, data types, and size limits against the API documentation.
5
Check Rate Limits
Verify you haven't exceeded rate limits. Check response headers for rate limit information.
6
Implement Retry Logic
For transient errors, implement exponential backoff retry with jitter. Check if error is retryable.
7
Search Error Catalog
Look up the specific error code in this catalog for detailed causes and solutions.
8
Contact Support
If the issue persists, contact support with error details, request ID, and troubleshooting steps taken.

Authentication Errors (AUTH_xxx)

Security 15 Errors

Authentication & Authorization

Errors related to API key validation, JWT tokens, OAuth flows, and permission checks.

AUTH_001
High

Invalid API Key

"The provided API key is invalid, expired, or has been revoked"
Common Causes
  • API key is malformed or contains invalid characters
  • API key has expired (default expiry: 1 year)
  • API key has been manually revoked or deactivated
  • Wrong environment API key (using testnet key in production)
  • API key not properly encoded in Authorization header
Solutions
  • Verify API key format matches pattern: nsl_[env]_[40_chars]
  • Check API key expiration date in dashboard
  • Generate new API key if current one is expired
  • Ensure using correct environment-specific API key
  • Verify Authorization header format: "Bearer [API_KEY]"
  • Check for trailing spaces or hidden characters
Correct API Key Usage
// Correct API key format const apiKey = 'nsl_prod_a1b2c3d4e5f6789012345678901234567890abcd'; // Correct authorization header const response = await fetch('https://api.neoservicelayer.com/storage/store', { method: 'POST', headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ key: 'test-key', value: 'test-value' }) }); // Check for authentication errors if (response.status === 401) { const error = await response.json(); if (error.code === 'AUTH_001') { console.error('Invalid API key - please check and regenerate'); } }
AUTH_002
High

Insufficient Permissions

"API key does not have required permissions for this operation"
Common Causes
  • API key created with read-only permissions
  • Attempting to access service not included in API key scope
  • Required role or permission level not assigned
  • Organization-level permissions not configured
  • API key restricted to specific IP addresses or domains
Solutions
  • Check API key permissions in dashboard settings
  • Request admin to upgrade API key permissions
  • Verify service is included in your subscription plan
  • Check IP whitelist settings if configured
  • Use service-specific API key with appropriate scope
  • Verify organization role assignments
Permission Check
// Check API key permissions before making requests const checkPermissions = async (apiKey) => { const response = await fetch('https://api.neoservicelayer.com/auth/permissions', { headers: { 'Authorization': `Bearer ${apiKey}` } }); const permissions = await response.json(); // Verify required permissions if (!permissions.services.includes('storage.write')) { throw new Error('Insufficient permissions for storage write operations'); } return permissions; }; // Use in your application try { await checkPermissions(apiKey); // Proceed with storage operations } catch (error) { console.error('Permission check failed:', error.message); }
AUTH_003
Medium

JWT Token Expired

"JSON Web Token has expired and must be refreshed"
Common Causes
  • JWT token exceeded its expiration time (default: 1 hour)
  • System clock skew between client and server
  • Token was issued with custom short expiration
  • Long-running processes with expired tokens
Solutions
  • Implement automatic token refresh mechanism
  • Check and synchronize system clock
  • Request new token using refresh token
  • Implement token expiry checks before API calls
  • Use longer-lived API keys for automated processes
Automatic Token Refresh
class AuthManager { constructor(apiKey, refreshToken) { this.apiKey = apiKey; this.refreshToken = refreshToken; this.token = null; this.tokenExpiry = null; } async getValidToken() { if (!this.token || this.isTokenExpired()) { await this.refreshAccessToken(); } return this.token; } isTokenExpired() { if (!this.tokenExpiry) return true; return Date.now() >= this.tokenExpiry - 60000; // Refresh 1 min before expiry } async refreshAccessToken() { const response = await fetch('https://api.neoservicelayer.com/auth/refresh', { method: 'POST', headers: { 'Authorization': `Bearer ${this.apiKey}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ refreshToken: this.refreshToken }) }); if (!response.ok) { throw new Error('Failed to refresh token'); } const data = await response.json(); this.token = data.accessToken; this.tokenExpiry = Date.now() + (data.expiresIn * 1000); } } // Usage const authManager = new AuthManager(apiKey, refreshToken); const token = await authManager.getValidToken();

Storage Errors (STORAGE_xxx)

Core Service 25 Errors

Storage Service Errors

Errors related to data storage, retrieval, encryption, replication, and quota management.

STORAGE_001
Medium

Invalid Key Format

"Storage key contains invalid characters or exceeds length limit"
Common Causes
  • Key contains special characters (only alphanumeric, hyphens, underscores allowed)
  • Key length exceeds 256 character limit
  • Key starts or ends with invalid characters
  • Key contains null bytes or control characters
  • Empty key provided
Solutions
  • Use only alphanumeric characters, hyphens (-), and underscores (_)
  • Ensure key length is between 1 and 256 characters
  • Remove or encode special characters
  • Use URL-safe encoding for complex identifiers
  • Implement key validation before API calls
Key Validation
// Key validation function function validateStorageKey(key) { if (!key || typeof key !== 'string') { throw new Error('Key must be a non-empty string'); } if (key.length > 256) { throw new Error('Key length exceeds 256 character limit'); } // Only allow alphanumeric, hyphens, and underscores const validKeyPattern = /^[a-zA-Z0-9_-]+$/; if (!validKeyPattern.test(key)) { throw new Error('Key contains invalid characters'); } return true; } // Usage examples const validKeys = [ 'user-profile-123', 'session_data_abc123', 'config-2024-01-30', 'user_settings_v2' ]; const invalidKeys = [ 'user profile 123', // Contains space 'config@settings', // Contains @ 'data/file/path', // Contains / '', // Empty string 'a'.repeat(257) // Too long ]; // Validate before storing try { validateStorageKey('user-profile-123'); await sdk.storage.store({ key: 'user-profile-123', value: userData }); } catch (error) { console.error('Key validation failed:', error.message); }
STORAGE_002
Medium

Value Too Large

"Data size exceeds the maximum allowed limit for single operation"
Common Causes
  • Data payload exceeds 1MB limit for single store operation
  • JSON serialization increases data size significantly
  • Binary data not properly encoded
  • Attempting to store large files without chunking
Solutions
  • Implement data chunking for large files
  • Use compression before storing
  • Store large data in multiple smaller operations
  • Use external storage with reference links
  • Optimize data structure to reduce size
Data Chunking Implementation
// Data chunking utility class StorageChunker { constructor(sdk, chunkSize = 900000) { // 900KB chunks (leaving room for metadata) this.sdk = sdk; this.chunkSize = chunkSize; } async storeLargeData(baseKey, data) { const serialized = JSON.stringify(data); const dataSize = new Blob([serialized]).size; if (dataSize <= this.chunkSize) { // Small enough for single operation return await this.sdk.storage.store({ key: baseKey, value: data }); } // Chunk the data const chunks = this.chunkData(serialized); const chunkKeys = []; // Store chunks for (let i = 0; i < chunks.length; i++) { const chunkKey = `${baseKey}_chunk_${i}`; await this.sdk.storage.store({ key: chunkKey, value: chunks[i] }); chunkKeys.push(chunkKey); } // Store metadata const metadata = { type: 'chunked_data', chunks: chunkKeys, totalSize: dataSize, createdAt: new Date().toISOString() }; return await this.sdk.storage.store({ key: baseKey, value: metadata }); } async retrieveLargeData(baseKey) { const result = await this.sdk.storage.get({ key: baseKey }); if (result.data.value.type !== 'chunked_data') { // Not chunked data, return as-is return result.data.value; } // Retrieve and combine chunks const metadata = result.data.value; const chunks = []; for (const chunkKey of metadata.chunks) { const chunkResult = await this.sdk.storage.get({ key: chunkKey }); chunks.push(chunkResult.data.value); } const combined = chunks.join(''); return JSON.parse(combined); } chunkData(data) { const chunks = []; for (let i = 0; i < data.length; i += this.chunkSize) { chunks.push(data.slice(i, i + this.chunkSize)); } return chunks; } } // Usage const chunker = new StorageChunker(sdk); // Store large data const largeData = { // ... large object }; try { await chunker.storeLargeData('large-dataset-123', largeData); console.log('Large data stored successfully'); } catch (error) { if (error.code === 'STORAGE_002') { console.error('Data too large even after chunking'); } }
