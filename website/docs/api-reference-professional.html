API Overview

The Neo Service Layer API is a RESTful API that provides access to 22+ blockchain services through a unified interface. All endpoints return JSON and use standard HTTP methods and status codes.

Base URLs

Production https://api.neo-service-layer.io/v1
Staging https://staging-api.neo-service-layer.io/v1
Development https://dev-api.neo-service-layer.io/v1

Authentication

All API requests require authentication using JWT Bearer tokens or API keys.

Authorization Header
Authorization: Bearer YOUR_JWT_TOKEN

Authentication

Neo Service Layer API uses JWT (JSON Web Tokens) for authentication. You can obtain tokens through our authentication endpoints or use API keys for server-to-server communication.

JWT Authentication

The primary authentication method for client applications.

POST
/auth/login
Authenticate user and receive JWT token

Request Body

application/json
{
  "email": "user@example.com",
  "password": "securePassword123",
  "blockchainType": "NeoN3"
}

Parameters

Parameter Type Required Description
email string ✅ User's email address
password string ✅ User's password
blockchainType enum ❌ Blockchain type: NeoN3 or NeoX

Success Response (200)

application/json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "rt_1234567890abcdef",
    "expiresIn": 3600,
    "user": {
      "id": "user_12345",
      "email": "user@example.com",
      "roles": ["user", "developer"],
      "permissions": ["storage:read", "storage:write", "oracle:read"]
    }
  },
  "meta": {
    "timestamp": "2024-01-30T12:00:00Z",
    "version": "v1.5.0",
    "requestId": "req_auth_123456"
  }
}

Error Response (401)

application/json
{
  "success": false,
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "Invalid email or password",
    "details": "The provided credentials do not match any user account"
  },
  "meta": {
    "timestamp": "2024-01-30T12:00:00Z",
    "version": "v1.5.0",
    "requestId": "req_auth_123457"
  }
}

cURL Example

bash
curl -X POST https://api.neo-service-layer.io/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123",
    "blockchainType": "NeoN3"
  }'

JavaScript Example

javascript
const response = await fetch('https://api.neo-service-layer.io/v1/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'securePassword123',
    blockchainType: 'NeoN3'
  })
});

const data = await response.json();

if (data.success) {
  // Store the token for future requests
  localStorage.setItem('authToken', data.data.token);
  console.log('Login successful:', data.data.user);
} else {
  console.error('Login failed:', data.error.message);
}

Python Example

python
import requests
import json

url = "https://api.neo-service-layer.io/v1/auth/login"
payload = {
    "email": "user@example.com",
    "password": "securePassword123",
    "blockchainType": "NeoN3"
}

response = requests.post(url, json=payload)
data = response.json()

if data['success']:
    token = data['data']['token']
    print(f"Login successful. Token: {token}")
else:
    print(f"Login failed: {data['error']['message']}")
Token Storage: Store JWT tokens securely and include them in the Authorization header for all subsequent API requests. Tokens expire after 1 hour by default.

Storage API

The Storage API provides decentralized data storage with encryption, access control, and versioning capabilities. Perfect for user data, application state, and content management.

Store Data

POST
/storage/store
Store data with optional encryption and access control

Request Body

application/json
{
  "key": "user-profile-123",
  "value": "{\"name\":\"Alice\",\"email\":\"alice@example.com\"}",
  "encrypted": true,
  "accessControl": {
    "public": false,
    "allowedAddresses": ["NX7dkCKKUHdFjG8gQJZxMdZLGKyKMzGrQs"],
    "permissions": ["read", "write"]
  },
  "metadata": {
    "contentType": "application/json",
    "category": "user-data",
    "tags": ["profile", "personal"]
  },
  "expiresAt": "2024-12-31T23:59:59Z"
}

Parameters

Parameter Type Required Description
key string ✅ Unique identifier for the data (max 256 chars)
value string ✅ Data to store (max 1MB)
encrypted boolean ❌ Whether to encrypt the data (default: false)
accessControl object ❌ Access control settings
metadata object ❌ Additional metadata for the data
expiresAt string ❌ ISO 8601 timestamp for data expiration

Success Response (201)

application/json
{
  "success": true,
  "data": {
    "transactionId": "0x1234567890abcdef",
    "blockHeight": 12345678,
    "gasUsed": "0.002",
    "storageHash": "sha256:abcdef1234567890",
    "createdAt": "2024-01-30T12:00:00Z",
    "expiresAt": "2024-12-31T23:59:59Z"
  },
  "meta": {
    "timestamp": "2024-01-30T12:00:00Z",
    "version": "v1.5.0",
    "requestId": "req_storage_123456"
  }
}

cURL Example

bash
curl -X POST https://api.neo-service-layer.io/v1/storage/store \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "user-profile-123",
    "value": "{\"name\":\"Alice\",\"email\":\"alice@example.com\"}",
    "encrypted": true,
    "metadata": {
      "contentType": "application/json",
      "category": "user-data"
    }
  }'
