Getting Started Tutorial

Learn the fundamentals of Neo Service Layer by building your first blockchain application. This hands-on tutorial covers environment setup, SDK basics, and core service integration.

30 min tutorial
Beginner Level
JavaScript/TypeScript

What You'll Build

In this tutorial, you'll create a simple data management application that demonstrates the core capabilities of Neo Service Layer. Your app will:

  • 🔐 Store encrypted data using the Storage Service
  • 🌐 Fetch external data using the Oracle Service
  • 📊 Display real-time updates with WebSocket integration
  • 🛡️ Handle authentication and error scenarios gracefully
Prerequisites: Basic JavaScript knowledge and Node.js installed on your system. No prior blockchain experience required!

Tutorial Goals

Learn Core Concepts

Understand Neo Service Layer architecture and service model

Hands-on Coding

Write real code with interactive examples and immediate feedback

Build Confidence

Deploy your first application and understand deployment patterns

Environment Setup

Let's prepare your development environment for building with Neo Service Layer.

1. Prerequisites Check

Ensure you have the following installed:

Check Node.js Version
node --version
npm --version

# Should show Node.js 14+ and npm 6+

2. Create New Project

Create a new directory and initialize a Node.js project:

Project Setup
mkdir neo-tutorial-app
cd neo-tutorial-app
npm init -y

# Install Neo Service Layer SDK
npm install @neo-service-layer/sdk

# Install development dependencies
npm install --save-dev @types/node typescript ts-node

3. Basic Project Structure

Create the basic project structure:

File Structure
neo-tutorial-app/
├── src/
│   ├── index.ts          # Main application file
│   ├── config.ts         # Configuration settings
│   └── services/         # Service integrations
├── package.json          # Project dependencies
├── tsconfig.json         # TypeScript configuration
└── .env                  # Environment variables

Create Your First Project

Now let's create the basic application structure and configuration files.

1. TypeScript Configuration

Create a tsconfig.json file for TypeScript settings:

tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

2. Environment Configuration

Create a .env file for environment variables:

.env
# Neo Service Layer Configuration
NSL_NETWORK=testnet
NSL_BLOCKCHAIN_TYPE=NeoN3
NSL_API_KEY=your-api-key-here

# Application Settings
APP_NAME=Neo Tutorial App
APP_VERSION=1.0.0

3. Configuration Module

Create src/config.ts to manage configuration:

// src/config.ts import { config } from 'dotenv'; // Load environment variables config(); export interface AppConfig { neoServiceLayer: { network: 'testnet' | 'mainnet'; blockchainType: 'NeoN3' | 'NeoX'; apiKey: string; endpoints?: { testnet?: { NeoN3?: string[]; NeoX?: string[]; }; mainnet?: { NeoN3?: string[]; NeoX?: string[]; }; }; }; app: { name: string; version: string; }; } export const appConfig: AppConfig = { neoServiceLayer: { network: (process.env.NSL_NETWORK as 'testnet' | 'mainnet') || 'testnet', blockchainType: (process.env.NSL_BLOCKCHAIN_TYPE as 'NeoN3' | 'NeoX') || 'NeoN3', apiKey: process.env.NSL_API_KEY || 'demo-key-for-testing', }, app: { name: process.env.APP_NAME || 'Neo Tutorial App', version: process.env.APP_VERSION || '1.0.0', }, }; // Validate configuration export function validateConfig(config: AppConfig): boolean { if (!config.neoServiceLayer.apiKey) { console.error('Missing NSL_API_KEY in environment variables'); return false; } if (!['testnet', 'mainnet'].includes(config.neoServiceLayer.network)) { console.error('Invalid NSL_NETWORK. Must be "testnet" or "mainnet"'); return false; } return true; } console.log('Configuration loaded:', { network: appConfig.neoServiceLayer.network, blockchainType: appConfig.neoServiceLayer.blockchainType, appName: appConfig.app.name, });

SDK Basics

Let's create the main application file and learn how to initialize the Neo Service Layer SDK.

1. Main Application Setup

Create src/index.ts as your main application entry point:

// src/index.ts import { NeoServiceLayerSDK } from '@neo-service-layer/sdk'; import { appConfig, validateConfig } from './config'; class TutorialApp { private sdk: NeoServiceLayerSDK | null = null; private isInitialized = false; async initialize(): Promise { try { // Validate configuration if (!validateConfig(appConfig)) { throw new Error('Invalid configuration'); } console.log('🚀 Initializing Neo Service Layer SDK...'); // Initialize SDK with configuration this.sdk = new NeoServiceLayerSDK(appConfig.neoServiceLayer); // Wait for SDK to connect await this.sdk.init(); this.isInitialized = true; console.log('✅ SDK initialized successfully!'); // Display connection info const networkInfo = await this.sdk.getNetworkInfo(); console.log('📡 Connected to network:', networkInfo); } catch (error) { console.error('❌ Failed to initialize SDK:', error); throw error; } } async run(): Promise { if (!this.isInitialized || !this.sdk) { throw new Error('App not initialized. Call initialize() first.'); } console.log('🎯 Starting tutorial application...'); // Run tutorial examples await this.runStorageExample(); await this.runOracleExample(); console.log('🎉 Tutorial completed successfully!'); } private async runStorageExample(): Promise { console.log('\\n📦 Storage Service Example:'); // We'll implement this in the next section } private async runOracleExample(): Promise { console.log('\\n🌐 Oracle Service Example:'); // We'll implement this in the next section } async cleanup(): Promise { if (this.sdk) { console.log('🧹 Cleaning up resources...'); await this.sdk.disconnect(); console.log('✨ Cleanup completed'); } } } // Application entry point async function main(): Promise { const app = new TutorialApp(); try { await app.initialize(); await app.run(); } catch (error) { console.error('💥 Application error:', error); process.exit(1); } finally { await app.cleanup(); } } // Handle graceful shutdown process.on('SIGINT', async () => { console.log('\\n👋 Graceful shutdown initiated...'); process.exit(0); }); // Run the application if (require.main === module) { main().catch((error) => { console.error('Fatal error:', error); process.exit(1); }); }
Try it out: Run the code above in the interactive playground to see how the SDK initialization works. The mock SDK will simulate real network connections.

Storage Service Example

Now let's implement the storage example to learn how to store and retrieve data securely.

Understanding the Storage Service

The Storage Service provides secure, encrypted data storage on the blockchain with features like:

  • Encryption: Automatic data encryption using AES-256
  • Access Control: Permission-based data access
  • Versioning: Track data changes over time
  • Metadata: Additional information about stored data

Implementation

Let's implement the storage example in our tutorial app:

// Update the runStorageExample method in src/index.ts private async runStorageExample(): Promise { console.log('\\n📦 Storage Service Example:'); try { // Example 1: Store simple data console.log('1️⃣ Storing user profile...'); const userProfile = { id: 'user-123', name: 'Alice Johnson', email: 'alice@example.com', preferences: { theme: 'dark', notifications: true }, createdAt: new Date().toISOString() }; const storeResult = await this.sdk!.storage.store({ key: 'user:profile:123', value: JSON.stringify(userProfile), encrypted: true, metadata: { contentType: 'application/json', version: '1.0', tags: ['user', 'profile'] } }); console.log('✅ Data stored successfully!'); console.log(' Transaction ID:', storeResult.transactionId); console.log(' Storage Key:', storeResult.key); // Example 2: Retrieve the stored data console.log('\\n2️⃣ Retrieving user profile...'); const retrieveResult = await this.sdk!.storage.get('user:profile:123'); const retrievedProfile = JSON.parse(retrieveResult.value); console.log('✅ Data retrieved successfully!'); console.log(' User Name:', retrievedProfile.name); console.log(' Email:', retrievedProfile.email); console.log(' Created:', retrievedProfile.createdAt); // Example 3: Update existing data console.log('\\n3️⃣ Updating user preferences...'); retrievedProfile.preferences.theme = 'light'; retrievedProfile.updatedAt = new Date().toISOString(); const updateResult = await this.sdk!.storage.update({ key: 'user:profile:123', value: JSON.stringify(retrievedProfile), metadata: { contentType: 'application/json', version: '1.1', tags: ['user', 'profile', 'updated'] } }); console.log('✅ Data updated successfully!'); console.log(' New Transaction ID:', updateResult.transactionId); // Example 4: List stored data console.log('\\n4️⃣ Listing user data...'); const listResult = await this.sdk!.storage.list({ prefix: 'user:', limit: 10 }); console.log('✅ Found stored items:'); listResult.items.forEach((item, index) => { console.log(` ${index + 1}. ${item.key} (${item.size} bytes)`); }); } catch (error) { console.error('❌ Storage example failed:', error); throw error; } }

Key Concepts Explained

🔑 Storage Keys

Use hierarchical keys like user:profile:123 for organization and easy querying.

🔐 Encryption

Enable encryption for sensitive data. The SDK handles encryption/decryption automatically.

📋 Metadata

Store additional information about your data for better organization and searchability.

🔍 Querying

Use prefixes and filters to efficiently find and retrieve related data items.

Oracle Service Example

Learn how to fetch external data and integrate real-world information into your blockchain application.

Understanding Oracles

Oracles bridge the gap between blockchain and real-world data. Neo Service Layer's Oracle Service provides:

  • Price Feeds: Real-time cryptocurrency and asset prices
  • Weather Data: Current weather conditions and forecasts
  • Sports Data: Live scores and statistics
  • Custom APIs: Connect to any external API

Implementation

// Update the runOracleExample method in src/index.ts private async runOracleExample(): Promise { console.log('\\n🌐 Oracle Service Example:'); try { // Example 1: Get cryptocurrency price console.log('1️⃣ Fetching Bitcoin price...'); const btcPriceRequest = await this.sdk!.oracle.requestData({ feed: 'crypto-price', parameters: { symbol: 'BTC', currency: 'USD' } }); console.log('✅ Bitcoin price data:'); console.log(' Price: $' + btcPriceRequest.data.price); console.log(' Timestamp:', btcPriceRequest.data.timestamp); console.log(' Confidence:', (btcPriceRequest.confidence * 100).toFixed(1) + '%'); // Example 2: Get weather data console.log('\\n2️⃣ Fetching weather data...'); const weatherRequest = await this.sdk!.oracle.requestData({ feed: 'weather', parameters: { city: 'New York', country: 'US' } }); console.log('✅ Weather data:'); console.log(' Temperature:', weatherRequest.data.temperature + '°C'); console.log(' Condition:', weatherRequest.data.condition); console.log(' Humidity:', weatherRequest.data.humidity + '%'); // Example 3: Subscribe to real-time updates console.log('\\n3️⃣ Subscribing to real-time price updates...'); const subscription = await this.sdk!.oracle.subscribe({ feed: 'crypto-price', parameters: { symbols: ['BTC', 'ETH', 'NEO'], currency: 'USD' }, interval: 30000 // Update every 30 seconds }); console.log('✅ Subscription created:'); console.log(' Subscription ID:', subscription.subscriptionId); console.log(' Feed:', subscription.feed); console.log(' Status:', subscription.active ? 'Active' : 'Inactive'); // Set up event listener for price updates this.sdk!.oracle.onData(subscription.subscriptionId, (data) => { console.log('📊 Price Update:', data); }); // Example 4: Custom API integration console.log('\\n4️⃣ Custom API example...'); const customApiRequest = await this.sdk!.oracle.requestData({ feed: 'custom-api', parameters: { url: 'https://api.github.com/repos/neo-project/neo', method: 'GET', headers: { 'User-Agent': 'Neo-Tutorial-App/1.0' } } }); console.log('✅ GitHub API data:'); console.log(' Repository:', customApiRequest.data.name); console.log(' Stars:', customApiRequest.data.stargazers_count); console.log(' Language:', customApiRequest.data.language); // Example 5: Batch data request console.log('\\n5️⃣ Batch data request...'); const batchRequest = await this.sdk!.oracle.batchRequest([ { feed: 'crypto-price', parameters: { symbol: 'BTC', currency: 'USD' } }, { feed: 'crypto-price', parameters: { symbol: 'ETH', currency: 'USD' } }, { feed: 'crypto-price', parameters: { symbol: 'NEO', currency: 'USD' } } ]); console.log('✅ Batch results:'); batchRequest.results.forEach((result, index) => { console.log(` ${result.parameters.symbol}: $${result.data.price}`); }); } catch (error) { console.error('❌ Oracle example failed:', error); throw error; } }
Note: The interactive playground uses mock data for demonstration. In a real application, you'll receive live data from actual oracle networks.
