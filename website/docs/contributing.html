Bug Reports

When reporting bugs, please provide as much detail as possible to help us reproduce and fix the issue quickly.

Bug Report Template

Required Information 
## Bug Description
Brief description of the bug

## Steps to Reproduce
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

## Expected Behavior
What you expected to happen

## Actual Behavior
What actually happened

## Environment
- Neo Service Layer version: [e.g. v3.0.0]
- OS: [e.g. Ubuntu 22.04, macOS 13.0, Windows 11]
- Node.js version: [e.g. 18.17.0]
- Browser: [e.g. Chrome 118, Safari 16] (if applicable)
- Additional dependencies: [e.g. Docker 24.0.0]

## Code Example (if applicable)
```javascript
// Code that reproduces the issue
const client = new NeoServiceLayer({...});
// ...
```

## Additional Context
Add any other context about the problem here.
Screenshots, logs, or error messages are very helpful.

Feature Requests

We welcome feature requests! Before submitting, please check if a similar request already exists and consider the following guidelines.

Feature Request Guidelines

Define the Problem

Clearly describe the problem you're trying to solve and why it's important.

Propose a Solution

Suggest how you think the feature should work. Include examples if possible.

Consider Alternatives

Think about alternative approaches and explain why your proposed solution is best.

Assess Impact

Consider the scope of changes needed and potential impact on existing functionality.

Feature Request Template

Template Structure 
## Problem Statement
What problem does this feature solve? Who would benefit from it?

## Proposed Solution
Describe your proposed solution in detail.

## Example Usage
```javascript
// Show how the feature would be used
const result = await client.newFeature({
    option1: 'value1',
    option2: 'value2'
});
```

## Alternatives Considered
What other approaches did you consider? Why is this approach better?

## Implementation Notes
- Any technical considerations
- Potential breaking changes
- Dependencies or prerequisites

## Additional Context
- Use cases
- Related issues or discussions
- External references

Coding Standards

JavaScript/TypeScript Standards 
// Use meaningful variable names
const blockchainTransactionProcessor = new TransactionProcessor();

// Add JSDoc comments for public APIs
/**
 * Retrieves a block by its hash
 * @param {string} blockHash - The hash of the block to retrieve
 * @param {boolean} includeTransactions - Whether to include transaction details
 * @returns {Promise} The block object
 * @throws {BlockNotFoundError} When block doesn't exist
 */
async function getBlockByHash(blockHash, includeTransactions = false) {
    validateBlockHash(blockHash);
    
    try {
        const block = await blockService.findByHash(blockHash);
        
        if (includeTransactions) {
            block.transactions = await transactionService.getByBlockHash(blockHash);
        }
        
        return block;
    } catch (error) {
        if (error.code === 'NOT_FOUND') {
            throw new BlockNotFoundError(`Block not found: ${blockHash}`);
        }
        throw error;
    }
}

// Use async/await instead of callbacks
// ✅ Good
const result = await processTransaction(transaction);

// ❌ Avoid
processTransaction(transaction, (error, result) => {
    // callback style
});

// Handle errors appropriately
try {
    const data = await apiCall();
    return data;
} catch (error) {
    logger.error('API call failed', { error: error.message, stack: error.stack });
    throw new ServiceError('Failed to fetch data', { cause: error });
}
Testing Standards 
// Use descriptive test names
describe('BlockService', () => {
    describe('getBlockByHash', () => {
        it('should return block with transactions when includeTransactions is true', async () => {
            // Arrange
            const blockHash = '0x1234...';
            const mockBlock = { hash: blockHash, height: 1000 };
            const mockTransactions = [{ hash: '0xabcd...' }];
            
            blockService.findByHash.mockResolvedValue(mockBlock);
            transactionService.getByBlockHash.mockResolvedValue(mockTransactions);
            
            // Act
            const result = await getBlockByHash(blockHash, true);
            
            // Assert
            expect(result).toEqual({
                ...mockBlock,
                transactions: mockTransactions
            });
            expect(transactionService.getByBlockHash).toHaveBeenCalledWith(blockHash);
        });
        
        it('should throw BlockNotFoundError when block does not exist', async () => {
            // Arrange
            const blockHash = '0x9999...';
            blockService.findByHash.mockRejectedValue(new Error('NOT_FOUND'));
            
            // Act & Assert
            await expect(getBlockByHash(blockHash))
                .rejects.toThrow(BlockNotFoundError);
        });
    });
});

Development Setup

Quick Setup 
# 1. Fork and clone the repository
git clone https://github.com/YOUR_USERNAME/neo-service-layer.git
cd neo-service-layer

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
# Edit .env.local with your configuration

# 4. Start development services
docker-compose -f docker-compose.dev.yml up -d

# 5. Run database migrations
npm run db:migrate

# 6. Start the development server
npm run dev

# 7. Run tests
npm test

# 8. Run linting
npm run lint

# 9. Build for production
npm run build
