Security Architecture

Intel SGX Enterprise Security

Intel SGX Enclaves

All sensitive operations run within hardware-protected execution environments that are isolated from the operating system and hypervisor.

  • Memory encryption at the hardware level
  • Remote attestation capabilities
  • Side-channel attack protection
  • Sealed storage for persistent data

Multi-Layer Authentication

Comprehensive authentication system supporting multiple methods for different security requirements.

  • API Keys with role-based permissions
  • JWT tokens with configurable expiration
  • OAuth 2.0 with PKCE support
  • Digital signature verification
  • Hardware wallet integration

Zero-Trust Architecture

Every request is authenticated and authorized, with no implicit trust between services.

  • Service-to-service mutual TLS
  • Network segmentation and isolation
  • Principle of least privilege
  • Continuous security monitoring

Compliance & Auditing

Built-in compliance features and comprehensive audit trails for regulatory requirements.

  • SOC 2 Type II compliance
  • GDPR privacy controls
  • Real-time audit logging
  • Immutable audit trails
  • Automated compliance reporting

store

POST Encrypted

Store data on the blockchain with optional encryption, metadata, and replication settings.

Method Signature
async store(params: StorageStoreParams): Promise
Parameters
Parameter Type Required Description
key string Required Unique identifier for the stored data (max 256 chars)
value string | object Required Data to store (max 1MB for single operation)
metadata StorageMetadata Optional Additional metadata including tags and custom fields
encryption EncryptionOptions Optional Encryption configuration (AES-256-GCM by default)
replication ReplicationOptions Optional Replication settings (3 replicas by default)
ttl number Optional Time to live in seconds (max 1 year)
tags string[] Optional Array of tags for categorization (max 10 tags)
Example Usage
// Store simple key-value data const result = await sdk.storage.store({ key: 'user-profile-123', value: { username: 'alice', email: 'alice@example.com', preferences: { theme: 'dark', notifications: true } }, metadata: { contentType: 'application/json', tags: ['user-data', 'profile'], customFields: { version: '1.0', source: 'web-app' } }, encryption: { algorithm: 'AES-256-GCM', encryptMetadata: false }, ttl: 86400 // 24 hours }); console.log('Stored successfully:', result.data); // Output: { key: 'user-profile-123', transactionId: 'tx_abc123...', blockIndex: 12345, ... } // Store with custom replication const sensitiveData = await sdk.storage.store({ key: 'private-key-backup', value: encryptedPrivateKey, encryption: { algorithm: 'ChaCha20-Poly1305', keyDerivation: 'Argon2id', encryptMetadata: true }, replication: { replicas: 5, regions: ['us-east', 'eu-west', 'asia-pacific'], consistency: 'strong', syncMode: 'sync' }, tags: ['backup', 'critical', 'encrypted'] });
Response Schema
interface StorageStoreResponse extends ServiceResponse { data: { key: string; // The storage key transactionId: string; // Blockchain transaction ID blockIndex: number; // Block number where stored storageHash: string; // Content hash for verification encryptionUsed: boolean; // Whether encryption was applied replicationStatus: { // Replication information totalReplicas: number; healthyReplicas: number; regions: string[]; }; estimatedRetrievalTime: number; // Expected retrieval time in ms }; }
Error Codes
Code Message Description Action
STORAGE_001 Invalid key format Key contains invalid characters or exceeds length limit Use alphanumeric characters, hyphens, and underscores only
STORAGE_002 Value too large Data exceeds 1MB size limit Reduce data size or use chunked storage
STORAGE_003 Key already exists Key is already in use Use different key or update existing entry
STORAGE_004 Insufficient storage quota Account storage limit exceeded Upgrade storage plan or delete unused data
STORAGE_005 Encryption failed Unable to encrypt data with provided options Check encryption parameters and try again

get

GET Sub-ms

Retrieve stored data by key with optional metadata and consistency level control.

Method Signature
async get(params: StorageGetParams): Promise
Example Usage
// Basic data retrieval const data = await sdk.storage.get({ key: 'user-profile-123', includeMetadata: true }); console.log('Retrieved data:', data.data.value); console.log('Last modified:', data.data.lastModified); // Strong consistency read const criticalData = await sdk.storage.get({ key: 'private-key-backup', consistencyLevel: 'strong', decryption: { verifyIntegrity: true } }); if (criticalData.success) { console.log('Securely retrieved:', criticalData.data.value); }

delete

DELETE Permanent

Permanently delete stored data and all its replicas from the blockchain.

Example Usage
// Delete data const result = await sdk.storage.delete({ key: 'user-profile-123', purgeReplicas: true, reason: 'User account deletion' }); if (result.success) { console.log('Data deleted successfully'); console.log('Transaction ID:', result.data.transactionId); }
