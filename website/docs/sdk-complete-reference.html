SDK Overview

The Neo Service Layer JavaScript/TypeScript SDK provides a comprehensive, type-safe interface for interacting with all blockchain services. Built with modern JavaScript and designed for production use, it offers complete TypeScript support, intelligent retry logic, connection pooling, and extensive error handling.

Full TypeScript Support

Complete type definitions, interfaces, and generic types for type-safe development

  • 500+ TypeScript interfaces
  • Generic type parameters
  • Strict null checks
  • Comprehensive JSDoc comments
  • IDE autocomplete support

Production Ready

Enterprise-grade reliability with comprehensive error handling and resilience patterns

  • Exponential backoff retry logic
  • Circuit breaker pattern
  • Connection pooling
  • Request timeout handling
  • Comprehensive error types

Real-time Features

WebSocket support for real-time blockchain events and data streaming

TypeScript Setup

The SDK includes complete TypeScript definitions. Configure your TypeScript project:

1. TypeScript Configuration

Add or update your tsconfig.json:

{ "compilerOptions": { "target": "ES2020", "module": "ESNext", "moduleResolution": "node", "lib": ["ES2020", "DOM"], "strict": true, "esModuleInterop": true, "skipLibCheck": true, "declaration": true, "declarationMap": true, "sourceMap": true, "resolveJsonModule": true, "allowSyntheticDefaultImports": true }, "include": ["src/**/*"], "exclude": ["node_modules", "dist"] }

2. Import with TypeScript

import { NeoServiceLayerSDK, SDKConfig, StorageService } from '@neo-service-layer/sdk'; // Full SDK instance with type safety const config: SDKConfig = { network: 'testnet', blockchainType: 'NeoN3', apiKey: process.env.NEO_API_KEY!, timeout: 30000, retryAttempts: 3 }; const sdk = new NeoServiceLayerSDK(config); // Service-specific imports for tree-shaking import { StorageService } from '@neo-service-layer/sdk/storage'; import { OracleService } from '@neo-service-layer/sdk/oracle'; const storage = new StorageService(config); const oracle = new OracleService(config);

Environment Setup

Configure your development environment for optimal SDK usage:

Development Environment

# Environment variables (.env.development) NEO_NETWORK=testnet NEO_BLOCKCHAIN_TYPE=NeoN3 NEO_API_KEY=your-development-api-key NEO_LOG_LEVEL=debug NEO_ENABLE_WEBSOCKETS=true NEO_RETRY_ATTEMPTS=3 NEO_TIMEOUT=30000 # Enable development features NEO_DEBUG_MODE=true NEO_VERBOSE_LOGGING=true NEO_MOCK_SERVICES=false

Production Environment

# Environment variables (.env.production) NEO_NETWORK=mainnet NEO_BLOCKCHAIN_TYPE=NeoN3 NEO_API_KEY=${NEO_PRODUCTION_API_KEY} NEO_LOG_LEVEL=error NEO_ENABLE_WEBSOCKETS=true NEO_RETRY_ATTEMPTS=5 NEO_TIMEOUT=15000 # Production optimizations NEO_DEBUG_MODE=false NEO_VERBOSE_LOGGING=false NEO_CONNECTION_POOLING=true NEO_CACHE_ENABLED=true

Testing Environment

# Environment variables (.env.test) NEO_NETWORK=testnet NEO_BLOCKCHAIN_TYPE=NeoN3 NEO_API_KEY=test-api-key NEO_LOG_LEVEL=silent NEO_ENABLE_WEBSOCKETS=false NEO_RETRY_ATTEMPTS=1 NEO_TIMEOUT=5000 # Testing configurations NEO_DEBUG_MODE=false NEO_VERBOSE_LOGGING=false NEO_MOCK_SERVICES=true NEO_FAST_FAIL=true
